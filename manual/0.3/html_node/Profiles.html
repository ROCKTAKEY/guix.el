3.2 Profiles

You can keep track of your Guix profiles with M-x guix-profiles command. By default, it displays only ~/.guix-profile and a system profile (on GuixSD). But if you manage multiple profiles, you can add them by setting guix-profiles variable, for example, like this: 

(with-eval-after-load 'guix-ui-profile
  (setq guix-profiles
        (append '("/path/to/my-profile1"
                  "/path/to/my-profile2")
                guix-profiles)))

Along with the general “list” keys (see List buffer), a “profile-list” buffer additionally provides the following bindings:

P

Display packages installed in profile at point.

G

Display generations of profile at point.

c

Make profile at point the current profile for the package commands (see Packages).
